# Plan 1: Change Detection Module

<objective>
Create `src/audits/changes.js` — the core diff engine for Phase 3. This module accepts two snapshots (a saved baseline and the latest pull) and produces a structured change report covering: new products, removed products, status changes, price changes, inventory changes, and inventory policy changes. Output is a readable terminal summary and a flat rows array suitable for CSV export. No CLI wiring — that is handled in Plan 2.
</objective>

<must_haves>
- Pure function: `diffSnapshots(baselineSnapshot, currentSnapshot)` — no file I/O, no API calls
- Compares by product `id` (Shopify GID, e.g., `gid://shopify/Product/123456`)
- Detects product-level changes: status, totalInventory, mediaCount, outOfStockBehavior
- Detects variant-level changes (per SKU): price, compareAtPrice, inventoryPolicy, inventoryQuantity
- Returns `{ summary, productChanges, variantChanges }` where each array is suitable for CSV export
- Prints a clear terminal change report (pastes cleanly into a client update)
- ESM module with named exports only
- Zero external dependencies (pure Node.js)
</must_haves>

The snapshot shape (from snapshot.js `loadLatestSnapshot`) is:
```
{
  pulledAt: string (ISO timestamp),
  counts: { products: number, variants: number, inventoryLevels: number },
  products: [{ id, title, handle, status, vendor, productType, tags,
               totalInventory, variantCount, mediaCount, hasDescription,
               outOfStockBehavior, internalId, updatedAt }],
  variants: [{ productId, productTitle, id, sku, barcode, price, compareAtPrice,
               inventoryQuantity, inventoryPolicy, weight, weightUnit, title,
               inventoryItemId, tracked }],
  inventoryLevels: [...]
}
```

Implement the following:

**Step 1 — Build lookup maps**

```js
// Index products by id
const baseProducts = new Map(baseline.products.map(p => [p.id, p]));
const currProducts = new Map(current.products.map(p => [p.id, p]));

// Index variants by sku (skip blank SKUs)
const baseVariants = new Map(baseline.variants.filter(v => v.sku).map(v => [v.sku, v]));
const currVariants = new Map(current.variants.filter(v => v.sku).map(v => [v.sku, v]));
```

**Step 2 — Detect product-level changes**

Loop over the union of all product IDs (from both maps):

For each product ID:
- If only in `currProducts` → it is NEW: push to `productChanges` with `changeType: 'added'`
- If only in `baseProducts` → it is REMOVED: push to `productChanges` with `changeType: 'removed'`
- If in both → compare these fields:
  - `status` — if changed: push with `changeType: 'status_change'`, include `from` and `to`
  - `totalInventory` — if changed: push with `changeType: 'inventory_change'`, include `from` and `to`
  - `outOfStockBehavior` — if changed: push with `changeType: 'policy_change'`, include `from` and `to`
  - `mediaCount` — if changed: push with `changeType: 'images_change'`, include `from` and `to`
  - A single product may generate multiple change rows (one per field changed)

Each `productChanges` row shape:
```js
{
  productId,
  title,
  handle,
  status,       // current status (or baseline if removed)
  changeType,   // 'added' | 'removed' | 'status_change' | 'inventory_change' | 'policy_change' | 'images_change'
  field,        // field name that changed (or '' for added/removed)
  from,         // baseline value ('' for added)
  to,           // current value ('' for removed)
}
```

**Step 3 — Detect variant-level changes**

Loop over the union of all SKUs (from both variant maps):

For each SKU:
- If only in `currVariants` → NEW variant: push with `changeType: 'variant_added'`
- If only in `baseVariants` → REMOVED variant: push with `changeType: 'variant_removed'`
- If in both → compare these fields per variant:
  - `price` — if changed: push with `changeType: 'price_change'`
  - `compareAtPrice` — if changed: push with `changeType: 'compare_price_change'`
  - `inventoryPolicy` — if changed: push with `changeType: 'policy_change'`
  - `inventoryQuantity` — if changed: push with `changeType: 'inventory_change'`

Each `variantChanges` row shape:
```js
{
  sku,
  productId,
  productTitle,
  changeType,
  field,
  from,
  to,
}
```

**Step 4 — Compute summary counts**

```js
const summary = {
  baselinePulledAt: baseline.pulledAt,
  currentPulledAt: current.pulledAt,
  productsAdded:   productChanges.filter(r => r.changeType === 'added').length,
  productsRemoved: productChanges.filter(r => r.changeType === 'removed').length,
  statusChanges:   productChanges.filter(r => r.changeType === 'status_change').length,
  inventoryChanges: productChanges.filter(r => r.changeType === 'inventory_change').length,
  policyChanges:   productChanges.filter(r => r.changeType === 'policy_change').length,
  imagesChanges:   productChanges.filter(r => r.changeType === 'images_change').length,
  variantsAdded:   variantChanges.filter(r => r.changeType === 'variant_added').length,
  variantsRemoved: variantChanges.filter(r => r.changeType === 'variant_removed').length,
  priceChanges:    variantChanges.filter(r => r.changeType === 'price_change').length,
  variantInventoryChanges: variantChanges.filter(r => r.changeType === 'inventory_change').length,
  variantPolicyChanges:    variantChanges.filter(r => r.changeType === 'policy_change').length,
  totalChanges:    productChanges.length + variantChanges.length,
};
```
