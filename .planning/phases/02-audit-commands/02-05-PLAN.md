# Plan 5: CLI Wiring + CSV Export for All Audits

<objective>
Wire all four audit modules (products, policy, discontinued, images) into `src/index.js` as CLI commands, and extend `src/csv.js` with per-audit CSV export functions. Each audit command follows the pattern: load latest snapshot → run audit function → print terminal output (already done inside the audit module) → export CSV to /exports/. This satisfies AUDIT-05 (CSV export for all audits) and completes the Phase 2 success criteria: `node src/index.js audit:products|audit:policy|audit:discontinued|audit:images`.
</objective>

<must_haves>
- All four audit commands wired into src/index.js switch statement
- Each command: loadLatestSnapshot → call audit fn → exportAuditCsv → done
- Graceful error if no snapshot found: print message and exit 1
- CSV column definitions for each audit type added to src/csv.js
- Each audit CSV saved to /exports/ with a descriptive name: `audit-products.csv`, `audit-policy-mismatches.csv`, `audit-discontinued.csv`, `audit-images.csv`
- CSV export uses the existing `exportCsv` internal function pattern in src/csv.js (add new named exports for each audit)
- Updated `default` case in src/index.js to list all new audit commands in the usage help text
- ESM module compatibility throughout
</must_haves>
