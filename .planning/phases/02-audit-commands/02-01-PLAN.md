# Plan 1: Product Health Audit

<objective>
Create `src/audits/products.js` that loads the latest pull snapshot and computes a comprehensive product health summary. The audit counts products by status (ACTIVE/DRAFT/ARCHIVED), measures field completeness percentages (description, productType, weight, barcode, SEO tags), and reports image and inventory policy distribution. Output is a formatted terminal table. This satisfies AUDIT-01 and is the foundation of `node src/index.js audit:products`.
</objective>

<must_haves>
- Reads from `loadLatestSnapshot('pull')` — no live API calls
- Counts: total products, ACTIVE, DRAFT, ARCHIVED
- Field completeness %: hasDescription, productType populated, mediaCount > 0
- Variant-level: weight > 0, barcode present, compareAtPrice present, inventoryPolicy = 'continue' vs 'deny'
- Prints a readable terminal summary table using `console.log` (no external table library required)
- Exports function `auditProducts(snapshot)` as named export — receives snapshot object, returns `{ summary, rows }`
- `rows` is an array of per-product objects suitable for CSV export (one row per product)
- Does NOT write any files — file export handled by the CLI wiring plan (02-05)
- ESM module (`export function auditProducts`)
</must_haves>

Create `/Users/srijan/Desktop/Rana App ITG/src/audits/products.js` as an ESM module.

The snapshot shape (from snapshot.js) is:
```
{
  pulledAt: string,
  counts: { products, variants, inventoryLevels },
  products: [{ id, title, handle, status, vendor, productType, tags, descriptionHtml, hasDescription,
               totalInventory, variantCount, mediaCount, outOfStockBehavior, internalId,
               createdAt, updatedAt, publishedAt }],
  variants: [{ productId, productTitle, id, sku, barcode, price, compareAtPrice,
               inventoryQuantity, inventoryPolicy, weight, weightUnit, title, inventoryItemId, tracked }],
  inventoryLevels: [{ inventoryItemId, sku, variantId, productId, productTitle,
                      locationId, locationName, available, onHand, incoming, committed }]
}
```

Implement `auditProducts(snapshot)`:

1. Build a variantsByProduct Map<productId, variant[]> from snapshot.variants for efficient lookup.

2. For each product in snapshot.products, compute:
   - status (ACTIVE / DRAFT / ARCHIVED — stored as uppercase string in snapshot)
   - hasDescription: snapshot field `hasDescription === 'yes'` (it's a string 'yes'/'no')
   - hasProductType: productType is non-empty string
   - hasImages: mediaCount > 0
   - For variants of this product:
     - anyWeightSet: any variant has weight > 0
     - anyBarcodeSet: any variant has barcode non-empty
     - anyCompareAtPrice: any variant has compareAtPrice non-empty
     - policyDeny: all variants have inventoryPolicy === 'DENY'
     - policyContinue: all variants have inventoryPolicy === 'CONTINUE'
     - policyMixed: some DENY, some CONTINUE

3. Accumulate summary counters:
   ```
   total, active, draft, archived,
   withDescription, withProductType, withImages,
   withWeight, withBarcode, withCompareAtPrice,
   allPolicyDeny, allPolicyContinue, mixedPolicy
   ```

4. Compute percentages: pct = (count / total * 100).toFixed(1)

5. Build `rows` array — one object per product with fields:
   ```
   { id, title, handle, status, hasDescription, hasProductType, mediaCount,
     anyWeightSet, anyBarcodeSet, anyCompareAtPrice, variantPolicySummary,
     totalInventory, outOfStockBehavior, updatedAt }
   ```
   where `variantPolicySummary` is 'all-deny' | 'all-continue' | 'mixed'

6. Print terminal summary using console.log with clear headers and alignment. Example format:
   ```
   === Product Health Audit ===
   Snapshot: 2026-03-19T14-30-00 | Products: 2482

   STATUS BREAKDOWN
     Active:    NNNN (NN.N%)
     Draft:     NNNN (NN.N%)
     Archived:  NNNN (NN.N%)

   FIELD COMPLETENESS
     Description:      NNN / NNNN (NN.N%)
     Product Type:     NNN / NNNN (NN.N%)
     Has Images:       NNN / NNNN (NN.N%)
     Weight Set:       NNN / NNNN (NN.N%)
     Barcode Set:      NNN / NNNN (NN.N%)
     Compare At Price: NNN / NNNN (NN.N%)

   INVENTORY POLICY
     All DENY:     NNNN (NN.N%)
     All CONTINUE: NNNN (NN.N%)
     Mixed:        NNNN (NN.N%)
   ```

7. Return `{ summary, rows }` where summary holds all the counters + percentages.

Export as:
```js
export function auditProducts(snapshot) { ... }
```
