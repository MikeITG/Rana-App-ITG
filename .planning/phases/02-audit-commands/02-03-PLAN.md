# Plan 3: Discontinued Products Audit

<objective>
Create `src/audits/discontinued.js` that identifies the 359 discontinued products still marked ACTIVE with zero total inventory. A product is considered "discontinued" if its `outOfStockBehavior` metafield contains "Remove item when out-of-stock" (indicating it should disappear when OOS) AND it is currently ACTIVE status AND totalInventory is 0 or less. These products should be Draft, not Active — they are blocking customers from navigating the catalogue and showing OOS pages. Satisfies AUDIT-03.
</objective>

<must_haves>
- Reads from snapshot — receives snapshot as parameter, no live API calls
- Discontinued detection criteria: status === 'ACTIVE' AND totalInventory <= 0 AND outOfStockBehavior contains "remove item" (case-insensitive)
- Secondary category: ACTIVE + totalInventory <= 0 + outOfStockBehavior contains "allow back" (backorderable but 0 stock — also problematic, 224 known)
- Exports `auditDiscontinued(snapshot)` returning `{ summary, rows }`
- `rows` shape: { productId, title, handle, status, totalInventory, outOfStockBehavior, issueType, tags, updatedAt }
- `issueType` values: 'DISCONTINUED_STILL_ACTIVE' | 'BACKORDERABLE_ZERO_STOCK'
- Prints terminal summary then lists first 20 discontinued products
- ESM module
</must_haves>

Detection logic:
- `status` in snapshot.products is uppercase: 'ACTIVE', 'DRAFT', 'ARCHIVED'
- `outOfStockBehavior` is the denormalized metafield value (may be empty string)
- `totalInventory` is a number (0 or positive; can be null — treat null as 0)
- `tags` is a comma-separated string (e.g., "CLEARANCE, FURNITURE")

Implement `auditDiscontinued(snapshot)`:

1. Initialize counters: discontinued = 0, backorderableZeroStock = 0, total = snapshot.products.length

2. For each product in snapshot.products:
   - Normalize: inv = product.totalInventory ?? 0
   - Normalize: oos = (product.outOfStockBehavior || '').toLowerCase()
   - isActive = product.status === 'ACTIVE'
   - isZeroStock = inv <= 0
   - isRemoveOOS = oos.includes('remove item')
   - isAllowBack = oos.includes('allow back')

   Classification:
   - If isActive && isZeroStock && isRemoveOOS → issueType: 'DISCONTINUED_STILL_ACTIVE', increment discontinued
   - Else if isActive && isZeroStock && isAllowBack → issueType: 'BACKORDERABLE_ZERO_STOCK', increment backorderableZeroStock
   - Else → skip (not a problem for this audit)

3. Build `rows` for all flagged products:
   ```
   {
     productId: product.id,
     title: product.title,
     handle: product.handle,
     status: product.status,
     totalInventory: inv,
     outOfStockBehavior: product.outOfStockBehavior,
     issueType,
     tags: product.tags,
     updatedAt: product.updatedAt
   }
   ```

4. Build summary:
   ```
   {
     total,
     activeProducts: count of products where status === 'ACTIVE',
     discontinued,          // ACTIVE + 0 stock + should-remove-OOS
     backorderableZeroStock, // ACTIVE + 0 stock + should-allow-backorders
     flagged: discontinued + backorderableZeroStock
   }
   ```

5. Print terminal output:
   ```
   === Discontinued Products Audit ===
   Snapshot: [pulledAt] | Total Products: NNNN

   RESULTS
     Active products total:              NNNN
     Discontinued but still ACTIVE:      NNNN  ← should be Draft (0 stock, remove-OOS metafield)
     Backorderable but 0 stock (ACTIVE): NNNN  ← policy allows backorders but nothing available
     Total flagged:                      NNNN

   DISCONTINUED STILL ACTIVE (first 20):
   [print first 20 DISCONTINUED_STILL_ACTIVE rows: Title | Handle | Inventory | OOS Behavior]
   ```

6. Return `{ summary, rows }`.

Export as:
```js
export function auditDiscontinued(snapshot) { ... }
```
