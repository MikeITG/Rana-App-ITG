# Plan 4: Image Audit

<objective>
Create `src/audits/images.js` that identifies products with insufficient images. The primary concern is ACTIVE products with 0 images (73 known) since these create a broken storefront experience. Secondary concerns are products with only 1 image and DRAFT/ARCHIVED products with 0 images (lower priority, but still useful data). The audit reads `mediaCount` from the snapshot's product rows, which was counted during the pull phase. Satisfies AUDIT-04.
</objective>

<must_haves>
- Reads from snapshot — receives snapshot as parameter, no live API calls
- `mediaCount` field on productRows is the source of truth (already computed in pull.js)
- Three tiers: ACTIVE with 0 images (critical), any status with 0 images, any status with exactly 1 image
- Exports `auditImages(snapshot)` returning `{ summary, rows }`
- `rows` shape: { productId, title, handle, status, mediaCount, issueType, updatedAt }
- `issueType` values: 'ACTIVE_NO_IMAGES' | 'DRAFT_NO_IMAGES' | 'ONE_IMAGE_ONLY'
- Summary covers counts by tier
- Terminal output groups results by severity (ACTIVE_NO_IMAGES first)
- ESM module
</must_haves>

Create `/Users/srijan/Desktop/Rana App ITG/src/audits/images.js` as an ESM module.

Detection logic:
- `mediaCount` is a number (0 or more) on each product in snapshot.products
- `status` is uppercase: 'ACTIVE', 'DRAFT', 'ARCHIVED'
- Priority order: ACTIVE_NO_IMAGES > DRAFT_NO_IMAGES > ONE_IMAGE_ONLY
- A product with 0 images only gets ONE_IMAGE_ONLY if it was somehow 0 AND not caught above — but practically: products with 0 images get a NO_IMAGES type, products with exactly 1 get ONE_IMAGE_ONLY

Implement `auditImages(snapshot)`:

1. For each product in snapshot.products:
   - count = product.mediaCount ?? 0
   - Classify:
     - count === 0 && status === 'ACTIVE' → issueType: 'ACTIVE_NO_IMAGES'
     - count === 0 && status !== 'ACTIVE' → issueType: 'DRAFT_NO_IMAGES' (covers DRAFT and ARCHIVED)
     - count === 1 → issueType: 'ONE_IMAGE_ONLY'
     - count >= 2 → no issue, skip

2. Build `rows` for all flagged products:
   ```
   {
     productId: product.id,
     title: product.title,
     handle: product.handle,
     status: product.status,
     mediaCount: count,
     issueType,
     updatedAt: product.updatedAt
   }
   ```

3. Build summary:
   ```
   {
     total: snapshot.products.length,
     activeNoImages: count,       // ACTIVE + 0 images — critical
     draftNoImages: count,        // DRAFT/ARCHIVED + 0 images
     oneImageOnly: count,         // exactly 1 image (any status)
     totalFlagged: activeNoImages + draftNoImages + oneImageOnly
   }
   ```

4. Print terminal output:
   ```
   === Image Audit ===
   Snapshot: [pulledAt] | Total Products: NNNN

   RESULTS
     Active products with 0 images:          NN  ← CRITICAL: broken storefront
     Draft/Archived products with 0 images:  NN
     Products with only 1 image:             NN
     Total flagged:                          NN

   ACTIVE PRODUCTS WITH NO IMAGES:
   [print all ACTIVE_NO_IMAGES rows: Title | Handle | Status | Image Count]

   PRODUCTS WITH ONLY 1 IMAGE (first 20):
   [print first 20 ONE_IMAGE_ONLY rows: Title | Handle | Status]
   ```

5. Return `{ summary, rows }`.

Export as:
```js
export function auditImages(snapshot) { ... }
```
