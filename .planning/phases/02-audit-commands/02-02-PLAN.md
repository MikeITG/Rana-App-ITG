# Plan 2: Inventory Policy Mismatch Audit

<objective>
Create `src/audits/policy.js` that loads the latest pull snapshot and detects inventory policy mismatches — the #1 critical issue in the store (853 products). The mismatch logic compares the `custom.out_of_stock_behavior_bp` metafield value (denormalized into `outOfStockBehavior` on productRows) against each variant's `inventoryPolicy`. Products where the metafield says "Allow back orders" but policy is DENY are flagged as WRONG. Products where metafield says "Remove item when out-of-stock" but policy is CONTINUE are also WRONG. Output is a terminal summary + per-product mismatch rows for CSV export. Satisfies AUDIT-02.
</objective>

<must_haves>
- Reads from snapshot — no live API calls; receives snapshot as parameter
- Mismatch logic must handle case-insensitive metafield value matching
- Two mismatch types: SHOULD_BE_CONTINUE (metafield says allow backorders, variant says deny) and SHOULD_BE_DENY (metafield says remove OOS, variant says continue)
- A third category: NO_METAFIELD — products where outOfStockBehavior is empty (unknown intent)
- Exports `auditPolicy(snapshot)` returning `{ summary, rows }` where rows are mismatch records only
- `rows` shape must include: productId, title, handle, status, sku, variantId, inventoryPolicy, outOfStockBehavior, mismatchType
- Prints terminal summary with counts, then lists up to 20 example mismatches
- ESM module
</must_haves>

Mismatch classification logic:
- `outOfStockBehavior` values from the metafield (case-insensitive matching):
  - Contains "allow back" → intended policy = CONTINUE (allow backorders)
  - Contains "remove item" → intended policy = DENY (remove when OOS)
  - Empty string → NO_METAFIELD (unknown intent)

- Variant `inventoryPolicy` from snapshot is stored as uppercase: 'DENY' or 'CONTINUE'

Implement `auditPolicy(snapshot)`:

1. Build a variantsByProduct Map<productId, variant[]> from snapshot.variants.

2. For each product in snapshot.products:
   - Get `outOfStockBehavior` (already denormalized on the product row)
   - Determine intended policy from metafield value (case-insensitive)
   - For each variant in variantsByProduct.get(product.id) || []:
     - If intended = CONTINUE and variant.inventoryPolicy === 'DENY':
       - Push mismatch row with mismatchType: 'SHOULD_BE_CONTINUE'
     - If intended = DENY and variant.inventoryPolicy === 'CONTINUE':
       - Push mismatch row with mismatchType: 'SHOULD_BE_DENY'
     - If NO_METAFIELD: push row with mismatchType: 'NO_METAFIELD' (track but separate)

3. Build `rows` array (mismatch records — all types):
   ```
   {
     productId, title, handle, status,
     sku, variantId, inventoryPolicy,
     outOfStockBehavior, mismatchType
   }
   ```

4. Compute summary:
   ```
   {
     total: snapshot.products.length,
     shouldBeContinue: count,   // "Allow back orders" but DENY — the critical 853
     shouldBeDeny: count,       // "Remove item" but CONTINUE
     noMetafield: count,        // no metafield value — unknown intent
     correct: count             // metafield matches policy
   }
   ```

5. Print terminal summary:
   ```
   === Inventory Policy Mismatch Audit ===
   Snapshot: [pulledAt] | Products: NNNN

   RESULTS
     Correct (metafield matches policy): NNNN
     MISMATCH — Should be CONTINUE:     NNNN  ← "Allow back orders" but policy=DENY
     MISMATCH — Should be DENY:         NNNN  ← "Remove item OOS" but policy=CONTINUE
     No OOS metafield (unknown intent): NNNN

   Total mismatches: NNNN

   SAMPLE MISMATCHES (first 20):
   [print first 20 rows as: SKU | Policy | OOS Behavior | Mismatch Type]
   ```

6. Return `{ summary, rows }` — rows includes ALL mismatch types (shouldBeContinue + shouldBeDeny + noMetafield), correct records are NOT included in rows.

Export as:
```js

```
